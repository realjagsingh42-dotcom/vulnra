Our Practices

How we secure VULNRA

Every security control we enforce on customer data. We don't hide behind vague assurances — here's the specific implementation.

🔐
Encryption at Rest
All scan results and user data stored in Supabase Postgres are encrypted at rest using AES-256. Redis cache entries use TLS-encrypted Upstash connections.
Live
🔒
Encryption in Transit
All API traffic enforces TLS 1.2+. HSTS headers set on all endpoints. Mixed content blocked. Certificate managed via Railway's auto-provisioned Let's Encrypt.
Live
🛡
Private IP Blocklist
Scan engine blocks RFC 1918 ranges (10.x, 172.16.x, 192.168.x), localhost (127.x), and cloud metadata endpoints (169.254.169.254) before any probe fires.
Live
🔑
API Key Scoping
API keys are scoped to account tier and enforced at middleware. Keys never appear in logs. Rotation available instantly from the dashboard. Keys use vk_ prefix for easy scanning.
Live
⏱
Data Retention Policy
Scan results are retained for 90 days maximum then purged. Target URLs are never passed to third-party analytics. No scan data is used for model training.
Live
🔍
Rate Limiting
All scan endpoints are rate-limited per API key using slowapi. Free: 1/day. Pro: 100/day. Enterprise: unlimited. Abuse patterns trigger automatic key suspension.
Live

Responsible Disclosure

How our disclosure programme works

We follow industry-standard coordinated disclosure. Here's the exact process from report to resolution.

01

You send a report

Email security@vulnra.ai with: vulnerability description, reproduction steps, affected URL or endpoint, impact assessment, and your contact details. PGP encryption available — see key below.

Day 0
02

We acknowledge receipt

We reply with a case number, confirmation of scope, and an initial severity assessment. We do not request that you wait indefinitely — you will hear from a human, not an autoresponder.

Within 24 hours
03

We investigate and reproduce

Our team reproduces the issue and assesses severity using CVSS 3.1 scoring. We keep you updated on progress. For critical vulnerabilities, we begin work immediately.

Days 1–14
04

We fix and verify

We deploy the fix and verify the vulnerability is resolved. We may ask you to retest if you're willing. We notify you of the fix date and planned disclosure timeline.

Days 7–45 depending on severity
05

Coordinated public disclosure

We publish a security advisory in the changelog and credit you by name (or anonymously if you prefer). Maximum disclosure timeline is 90 days from initial report.

Day 90 maximum

Bug bounty programme

We offer recognition and rewards for valid, in-scope vulnerability reports. We do not offer monetary rewards yet — that changes when we hit $10K MRR.

Severity Examples Reward

FAQ

Security questions

Does VULNRA store the content of my LLM's responses? ▾
VULNRA stores the probe results (pass/fail status, risk score, finding categories) for 90 days. The raw LLM response text is retained for up to 7 days to support PDF report generation, then purged. Response content is never passed to third-party analytics or used for model training.
Can VULNRA scans be detected by the target LLM? ▾
Yes — VULNRA scans appear as API traffic to the target endpoint. We do not attempt to disguise probe traffic. If you are scanning a production endpoint, ensure you have authorisation and that rate limits won't be triggered by the scan volume. Pro tier scans run approximately 40 API calls against the target.
Is my API key to the target LLM ever logged or stored? ▾
VULNRA does not ask for or store API keys to your target LLM endpoint. You provide the endpoint URL. If your endpoint requires authentication, you should configure VULNRA to use a dedicated test key with minimal permissions — not your production key.
Can I scan a competitor's LLM with VULNRA? ▾
No. VULNRA's Acceptable Use Policy requires that you only scan endpoints you own or have explicit written authorisation to test. Scanning third-party services without authorisation violates our AUP and may violate computer misuse laws. Violations result in immediate account termination.
When will VULNRA achieve SOC 2 Type II certification? ▾
We are currently in the evidence collection phase using Vanta. We are targeting Q3 2026 for SOC 2 Type II certification. Enterprise customers requiring SOC 2 before that date should contact us directly — we can share our current control documentation under NDA.
